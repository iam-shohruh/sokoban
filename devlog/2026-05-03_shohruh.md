## What I did

### Implemented A* solver
`a_star.py` was an empty stub. I filled it in with a full A* implementation:
- Priority queue (min-heap) ordered by `f = g + h`
- `g` = number of pushes so far
- `h` = `manhattan_heuristic`: sum of greedy nearest-goal Manhattan distances per box
- Deadlock pruning via `env.is_deadlock()` at each expansion
- `debug` / `steps_out` modes to capture every expanded state for the replay viewer

### Built the replay viewer (`viz/replay.py`)
Interactive pygame window that lets you step through every state A* explored:
- Left/Right arrows to step manually, Space to play/pause, R to restart, Up/Down to change speed
- HUD shows current step, push direction, and `g / h / f` values
- **Live mode**: runs the solver in a background thread and streams states into the viewer in real time as the search runs (`visualize_live`)
- Can also replay an already-collected step list (`visualize_replay`)
- Run directly: `python -m sokoban.viz.replay --dataset microban --game_id 1`

## What I found out

A* solves microban levels 1ŌĆō92 fine. **It gets stuck on level 93.**

Level 93 has 8 boxes (compared to 3ŌĆō4 in surrounding levels). The state space explodes.

Two bugs in the current heuristic make this worse:

1. **Inadmissible**: the greedy nearest-goal assignment can overestimate the true cost. Example: if Box1 is distance [1, 2] from [Goal1, Goal2] and Box2 is [1, 3], greedy assigns Box1ŌåÆGoal1 and gives total=4, but the optimal assignment is total=3. An overestimating heuristic causes A* to skip the optimal path.

2. **Loose**: Manhattan distance ignores walls. The true minimum number of pushes to move a box across a maze is often much higher.

Added stubs for `hungarian_heuristic`, `pattern_database_heuristic`, and `hybrid_heuristic` ŌĆö not implemented yet.

## Next

Replace `manhattan_heuristic` with a proper admissible heuristic:
- **Precompute push distances**: BFS backwards from each goal using pull moves (ignoring player repositioning) to get true minimum pushes from any cell to that goal, accounting for walls.
- **Optimal boxŌĆōgoal assignment**: bitmask DP over the precomputed distances (`O(n┬▓ ├Ś 2Ōü┐)`), which is fast for `n Ōēż 15` and correct (not greedy).

This is the standard approach in competitive Sokoban solvers and should unblock level 93.
