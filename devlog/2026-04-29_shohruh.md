**Decisions:**
- the puzzle has static and dynamic components:
    - static: goals, walls, spaces
    - dynamic: player, boxes
- instead of storing everything in every new state, we'll just store what's changing - dynamic components
    - State dataclass will store player position as tuple and box positions as a set of tuples
    - Level dataclass will store goals and walls as tuple and store initial State. 
    - in each iteration of search, State will be updated and saved to check if we've been in that state in earlier episodes of search.
