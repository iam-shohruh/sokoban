## What changed

### `state.py`
- `Position` and `Push` converted from `typing.Tuple` aliases to proper `NamedTuple` classes
  - `Position(row, col)` — row-first ordering to match grid iteration and env move deltas
  - `Push(position, direction)` — typed, instantiable, still tuple-compatible
- `Level.state` renamed to `Level.init_state` to make it clear it's the starting state, not the current one
- Removed `typing.Tuple` import, replaced with `typing.NamedTuple`

### `env.py`
- Split `SokobanEnv` into two classes:
  - `SokobanRules` — stateless, owns all game logic (`step`, `get_valid_pushes`, `is_goal_state`)
  - `SokobanEnv` — thin wrapper around a level, delegates to `SokobanRules`
- `get_valid_actions` (player-move-based) replaced by `get_valid_pushes` (BFS over reachable player positions to find all valid box pushes)
- `step` now takes explicit `state` and `push` arguments instead of mutating `self.current_state`
- All positions and pushes now typed as `Position` / `Push` instead of raw tuples

### `search/bfs.py`
- Fixed path reconstruction: replaced incorrect linear append with a `parent` map (state → (prev\_state, action)); backtracks from goal to start and reverses
- Changed from `level.state` to `level.init_state`
- Changed from `get_valid_actions` to `get_valid_pushes`
- Solver is now stateless — passes state explicitly through each BFS step

### `eval.py`
- Fixed dataset paths: `"sokoban/data/parsed/..."` → `"data/parsed/.../levels.json"`
- Fixed level loading: `game_id` key → `id` (matches actual JSON schema)
- Fixed coordinate order: `Position(i, j)` → `Position(j, i)` so row comes first, consistent with `Position(row, col)`
- Fixed `Level` construction: `state=` → `init_state=`

### `pyproject.toml` (new)
- Added minimal packaging config so the `sokoban` package can be installed with `pip install -e .` and imported from anywhere

## What's done
- BFS solver runs end-to-end on microban dataset
- Environment correctly enumerates all valid pushes via reachability BFS
- State representation is clean, typed, and hashable throughout

## Decisions

- **`SokobanRules` vs `SokobanEnv` separation** — rules are stateless and level-agnostic; env binds rules to a specific level. This keeps game logic testable independently of any dataset.
- **`Position` as `NamedTuple` with `(row, col)` ordering** — row-first matches how the grid is iterated (`for j, row`) and how move deltas are defined in env (`UP: (-1, 0)`). Using named fields prevents silent coordinate swaps.
- **Parent map for BFS path** — a linear append to a path list doesn't work in BFS because multiple branches are explored simultaneously. The parent map correctly reconstructs the unique path from start to goal.
- **`init_state` instead of `state` on `Level`** — `Level` is static; calling the field `state` implied mutability. `init_state` makes the intent explicit.
